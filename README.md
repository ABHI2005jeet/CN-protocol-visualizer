# 📡 CN Protocol Visualizer
### Interactive Simulation of Controlled Access Protocols in Computer Networks

An **interactive web-based visualizer** designed to help students understand **Controlled Access Protocols** in Computer Networks through **simulation, theory, and numerical demonstrations**.

This project focuses on three fundamental MAC layer protocols:

- Polling Protocol
- Voting Protocol
- Reservation Protocol

The platform combines **visual simulations, theoretical explanations, and numerical calculators** to provide a complete learning experience for students studying **Data Communications and Networking**.

# 📚 Protocols Covered

## 1️⃣ Polling Protocol

Polling is a **centralized controlled access mechanism** where a **master node sequentially checks each station** to determine if it has data to transmit.

### Key Characteristics
- Master–Slave architecture  
- Collision-free transmission  
- Sequential node checking  

### Example

If:

Number of nodes = n  
Time per node = t  

Total polling delay:

Delay = n × t

### Real-Life Analogy

Teacher calling students one-by-one during attendance.

---

## 2️⃣ Voting Protocol

Voting protocol is used in **distributed systems** to maintain consistency using a **quorum mechanism**.

### Key Characteristics

- Distributed decision making  
- Quorum-based validation  
- Fault tolerance  

### Quorum Rule

q > n / 2

Where:

q = quorum  
n = total nodes

### Real-Life Analogy

Election voting system where majority decides.

---

## 3️⃣ Reservation Protocol

Reservation protocol allows stations to **reserve time slots before transmission**, ensuring collision-free communication.

### Key Characteristics

- Time divided into frames  
- Stations reserve transmission slots  
- Efficient bandwidth allocation  

### Efficiency Formula

Efficiency = Used Slots / Total Slots

### Real-Life Analogy

Train seat reservation system.
