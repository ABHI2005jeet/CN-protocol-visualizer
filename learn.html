Polling Protocol

📌 Numerical Nature

Polling is mostly conceptual.

Basic formula:

Delay = n × t

No complex derivations are involved.

Definition: Polling is a controlled access mechanism in which a central controller (master) asks each node sequentially whether it has data to transmit.

Relevance in CN: It belongs to Controlled Access protocols in MAC layer and avoids collisions.

Working:

  • Master node sends request to node 1
  • If node has data → it transmits
  • Else → master moves to next node
  • Cycle continues for all nodes

Real Life Example: Teacher calling students one by one.

Example 1: 4 nodes, each takes 2 ms → Total delay = 8 ms

Example 2: If node 3 has data, it must wait for its turn → delay increases.

Advantages:

  • No collision
  • Simple

Disadvantages:

  • High delay
  • Idle polling wastes time

Voting Protocol

📌 Numerical Nature

Voting is purely logical and conceptual.

Based on quorum rule:

q > n / 2

No heavy calculations are required.

Definition: Voting protocol is used to maintain consistency in distributed systems using quorum.

Relevance in CN: Used in distributed databases and consistency control.

Working:

  • Each node casts a vote
  • If votes ≥ quorum → operation allowed
  • Else → denied

Real Life Example: Election system.

Example 1: 5 nodes, quorum = 3 → valid system

Example 2: 5 nodes, quorum = 2 → unsafe system

Advantages:

  • Ensures consistency
  • Fault tolerant

Disadvantages:

  • Overhead in voting
  • Complex

Reservation Protocol

📌 Numerical Nature

Reservation has slight numerical aspect.

Efficiency formula:

Efficiency = Used Slots / Total Slots

It is still a simple ratio-based calculation.

Definition: Reservation protocol allows stations to reserve time slots before transmission.

Relevance in CN: Controlled Access protocol in MAC layer.

Working:

  • Time divided into frames
  • Each station reserves slot
  • Transmission happens in reserved slot

Real Life Example: Train seat booking.

Example 1: 4 slots, 2 used → efficiency = 50%

Example 2: All slots used → efficiency = 100%

Advantages:

  • No collision
  • Efficient usage

Disadvantages:

  • Idle slots waste bandwidth
